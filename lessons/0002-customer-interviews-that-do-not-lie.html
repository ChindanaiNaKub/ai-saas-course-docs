Lesson 0002 · Customer Discovery

คุยกับลูกค้าให้ได้ความจริง ไม่ใช่คำชม

บทเรียนนี้สอนทักษะเดียว: เปลี่ยนคำถามที่ทำให้เราหลงดีใจ เป็นคำถามที่ดึงพฤติกรรมจริง, cost จริง, และ commitment จริงออกมา.

คำชมที่อันตราย

"น่าสนใจนะ", "ทำเสร็จแล้วส่งมาดู", "ถ้ามีแบบนี้ก็ดี" ฟังดี แต่ยังไม่แปลว่ามี demand.

หลักฐานที่แข็งกว่า

เหตุการณ์ล่าสุด, workaround ปัจจุบัน, เงิน/เวลาที่เสีย, budget owner, ข้อมูลจริง, นัดต่อ, pilot, หรือ intro.

1. ทำไม customer interview ถึงหลอก founder เก่ง?

เพราะคนส่วนใหญ่สุภาพ. โดยเฉพาะบริบทไทย หลายคนไม่อยากหักหน้า ไม่อยากพูดว่าไอเดียเราไม่สำคัญ หรือไม่อยากเสียบรรยากาศ. ถ้าเราถามว่า "คุณชอบไหม?" เรากำลังเชิญให้เขาตอบแบบรักษาน้ำใจ.

ดังนั้นเป้าหมายไม่ใช่การ pitch. เป้าหมายคือการสืบสวนชีวิตจริงของเขา: เขาทำงานยังไง, ปัญหาเกิดเมื่อไหร่, เขาแก้ยังไง, เขาเสียอะไร, และเรื่องนี้สำคัญพอไหมเมื่อเทียบกับปัญหาอื่น.

อย่าถามว่าเขาจะใช้ product ของเราไหม. ให้ถามว่าเขาเพิ่งเจอปัญหานี้จริงไหม และตอนนี้เขาทำอะไรกับมันอยู่.

แหล่งอ้างอิงหลักของบทเรียนนี้คือ The Mom Test และ summary โดย Sachin Rekhi: ถามเรื่องชีวิตของลูกค้า ไม่ถามความเห็นเกี่ยวกับไอเดียของเรา.

2. คำถาม 3 แบบ: อันตราย, พอใช้, ดี

แบบตัวอย่างผลลัพธ์ที่มักได้
อันตราย "ถ้าผมทำ SaaS ช่วยสรุปแชท LINE คุณจะจ่ายไหม?" ได้คำตอบสมมติที่ไม่มีต้นทุน: "อาจจะ", "น่าสนใจ"
พอใช้ "คุณมีปัญหาตอบแชทลูกค้าช้าไหม?" เริ่มแตะ pain แต่ยังนำคำตอบและกว้างเกินไป
ดี "สัปดาห์ที่แล้วมีเคสไหนที่แชทลูกค้าหลุดหรือปิดการขายไม่ทันไหม เล่าให้ฟังหน่อยว่าเกิดอะไรขึ้น" ได้เหตุการณ์จริง, context, cost, และ workflow

3. Interview script 20 นาที

ใช้ script นี้เมื่อคุณยังไม่มี product หรือมีแค่ idea/manual demo. อย่าเปิดด้วย deck ยาว เปิดด้วยความอยากเข้าใจงานของเขา.

  1. Set frame: "ผมยังไม่ได้ขายอะไร วันนี้อยากเข้าใจ workflow จริง ถ้าไม่ใช่ปัญหาก็บอกตรงๆ ได้เลย"
  2. Workflow: "ตอนนี้งานนี้เริ่มจากตรงไหน จบตรงไหน ใครเกี่ยวข้องบ้าง"
  3. Recent event: "ครั้งล่าสุดที่มันพัง/ช้า/แพง/หลุด เกิดเมื่อไหร่"
  4. Cost: "ตอนนั้นเสียเวลา เงิน ลูกค้า หรือความเสี่ยงประมาณเท่าไหร่"
  5. Workaround: "ตอนนี้ใช้วิธีอะไรแก้ ใช้ Excel, LINE, คน, agency, หรือ tool อะไร"
  6. Priority: "เดือนนี้เรื่องนี้ติด top 3 ที่อยากแก้ไหม ถ้าไม่ใช่ อะไรสำคัญกว่า"
  7. Commitment: "ถ้าผมลองทำ manual version ให้ดู 1 รอบ คุณยอมส่งตัวอย่างข้อมูล/นัด follow-up/แนะนำอีก 2 คนไหม"

ข้อสุดท้ายสำคัญมาก เพราะ YC เน้นว่า early founders ต้อง recruit users เองและเรียนจากของจริง (Do Things That Don't Scale). Commitment แยกคนที่แค่สุภาพออกจากคนที่มี pain จริง.

4. อ่าน signal หลังคุย

Build signal

  • มีเหตุการณ์ล่าสุดภายใน 30 วัน
  • เขาเสียเงิน/เวลา/โอกาสชัดเจน
  • เขามี workaround ที่น่ารำคาญหรือแพง
  • เขายอมส่งข้อมูลจริงหรือนัดต่อ
  • เขาแนะนำคนอื่นที่เจอปัญหาเดียวกัน

Do not build yet

  • เขาพูดแต่ "น่าสนใจ" แต่ไม่มีเรื่องจริง
  • ปัญหาเกิดนานๆ ทีและไม่มี cost
  • เขาไม่เคยพยายามแก้ด้วยวิธีไหนเลย
  • เขาไม่ยอมเสียเวลา 15 นาทีต่อ
  • เขาอยากได้ feature เยอะ แต่ไม่เห็น urgency

5. แบบฝึกหัด: แปลงคำถามให้ดีขึ้น

พิมพ์คำถามที่คุณอยากถามลูกค้า แล้วกดประเมิน. เป้าหมายคือให้คำถามพูดถึงอดีต, เหตุการณ์จริง, workflow, cost, หรือ workaround.

รอคำถามของคุณอยู่.

6. แบบฝึกหัด: สรุป interview note เป็น signal

หลังคุยจริง ให้เอา note มาวางตรงนี้ หน้าเว็บจะช่วยเตือนว่า note มี signal แข็งพอไหม.

รอ note ของคุณอยู่.

7. Mini Quiz

ลูกค้าพูดว่า: "ไอเดียดีนะ ถ้าทำเสร็จแล้วส่งมาให้ดูได้เลย"

คุณควรตอบยังไง?

เลือกคำตอบหนึ่งข้อ.

8. Homework ก่อนบทเรียน 3

เลือก niche เดียวจากบทเรียน 1 แล้วทำ interview 3 คน โดยไม่ pitch product ก่อน. หลังคุย ให้จดเป็น 5 บรรทัดต่อคน:

  1. ลูกค้าคือใคร
  2. เหตุการณ์ล่าสุดที่เจอปัญหา
  3. cost ที่เสีย
  4. workaround ปัจจุบัน
  5. commitment ที่เขายอมให้

เอา raw notes กลับมาส่งผมได้เลย. ผมจะช่วย classify ว่าอันไหนเป็น signal จริง, อันไหนเป็นคำชม, และควร build/manual test อะไรต่อ.
