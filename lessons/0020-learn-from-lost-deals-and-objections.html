Lesson 0020 · Lost deal learning loop

เปลี่ยนดีลที่แพ้ให้เป็นข้อมูลที่ทำให้ขายดีขึ้น

บทเรียนนี้สอนทักษะเดียว: classify objections และ lost reasons ให้ถูก เพื่อรู้ว่าควรแก้ ICP, message, trust, urgency, package, price, หรือ timing.

กับดัก

ลูกค้าบอกว่า "ขอคิดดูก่อน" หรือ "แพงไป" แล้วคุณรีบลดราคา/build feature ทั้งที่เหตุผลจริงอาจเป็น pain ไม่แรง, buyer ไม่ใช่คนซื้อ, หรือ trust ยังไม่พอ.

เป้าหมาย

ทุกดีลที่แพ้ต้องจบด้วย learning หนึ่งข้อและ experiment ถัดไปหนึ่งอย่าง ไม่ใช่จบด้วยความรู้สึกว่าตลาดไม่เอา.

1. Objection ไม่ใช่เหตุผลจริงเสมอไป

ใน founder-led sales คุณไม่ได้แค่พยายาม close. คุณกำลังเรียนรู้ว่าตลาดซื้ออะไร ไม่ซื้ออะไร และทำไม. YC sales playbook และ First Round founder-led growth เน้นว่าการขายช่วงแรกคือ feedback loop ที่ founder ต้องอยู่ใกล้มากพอ. ถ้าคุณจัดหมวด lost reason ผิด คุณจะปรับ product/ราคา/message ผิดทาง.

คำว่า "แพงไป" บางครั้งแปลว่าราคาแพงจริง. แต่บ่อยครั้งแปลว่า value, urgency, trust, หรือ buyer fit ยังไม่ชัด.

2. Lost reason 7 แบบ

หมวดหน้าตาสิ่งที่ควรแก้
ICP fit เขาสนใจ แต่ workflow/budget/scale ไม่ตรงกับ package ปรับ targeting ไม่ใช่ product
Pain weak เขาบอกว่าน่าสนใจ แต่ไม่มี cost ของปัญหาชัด ถาม past behavior และหา segment ที่เจ็บกว่า
Trust gap กังวล data, accuracy, security, proof, หรือคุณยังดูเสี่ยง เพิ่ม proof, pilot, guarantee, data handling
Urgency gap เห็นด้วยแต่เลื่อนไปเรื่อยๆ หา trigger/event/deadline ที่ทำให้ต้องแก้ตอนนี้
Package mismatch อยากได้ outcome แต่ scope/usage/onboarding ไม่พอดี ปรับ package หรือ ladder ไม่ใช่ลดมั่วๆ
Price/value gap เข้าใจ value แล้ว แต่ ROI หรือ budget ไม่ผ่าน ปรับ value framing, price metric, buyer segment
Timing fit ดี แต่ติดรอบงบ, migration, season, internal priority follow-up date และ nurture proof

3. ถามหลังแพ้ดีลให้ได้ความจริง

อย่าถามว่า "ทำไมไม่ซื้อครับ" แบบกดดัน. ถามให้เขาช่วย calibrate คุณ.

คำถามดี

  • "อะไรคือเหตุผลหลักที่ยังไม่ใช่ตอนนี้?"
  • "ถ้าต้องเลือกหนึ่งอย่างที่ทำให้ลังเลที่สุด คืออะไร?"
  • "ปัญหานี้มี cost จริงประมาณไหนในเดือนที่ผ่านมา?"
  • "ใครต้องเห็น/อนุมัติก่อนถึงจะเดินต่อได้?"

คำถามที่ทำให้ได้คำตอบปลอม

  • "ถ้าถูกกว่านี้จะซื้อไหม?"
  • "ชอบ product ไหม?"
  • "ถ้าเพิ่ม feature นี้จะซื้อไหม?"
  • "คิดว่า idea นี้ดีไหม?"

4. แพทเทิร์นสำคัญกว่าเคสเดียว

ดีลเดียวแพ้อาจเป็น noise. แต่ถ้า 5 ดีลแพ้ด้วยเหตุผลเดียวกัน นั่นคือ signal. ให้ review lost reasons ทุกวันศุกร์คู่กับ pipeline metrics จากบท 19.

3+ ICP fit losses

lead source หรือ targeting ผิด. กลับไป sharpen ICP/positioning.

3+ Trust losses

ต้องเพิ่ม proof, data handling, case study, pilot design, หรือ guarantee.

3+ Price losses

เช็ก value metric, budget owner, package scope, และ segment willingness-to-pay.

5. แบบฝึกหัด: Lost reason classifier

กรอกเหตุการณ์ดีลที่แพ้ แล้วเลือกสัญญาณ. ตัวประเมินนี้เป็น JavaScript ในไฟล์ ไม่เรียก AI และไม่ส่งข้อมูลออกไปไหน.

รอข้อมูลดีลอยู่.

6. ร่าง post-loss follow-up

ใช้ส่งหลังดีลไม่ไปต่อ เพื่อขอเรียนรู้โดยไม่กดดัน.

กรอกข้อมูลแล้วกดร่างข้อความ.

7. Mini Quiz

สถานการณ์: Prospect 5 รายตอบว่า "แพงไป" แต่ทั้ง 5 รายยังไม่เคยวัด cost ของปัญหา และ 4 รายเป็นพนักงานที่ไม่มี budget authority. คุณควรสรุปว่าอะไร?

เลือกคำตอบหนึ่งข้อ.
