Lesson 0014 · Customer health signals

กัน churn ก่อนลูกค้าหายเงียบ

บทเรียนนี้สอนทักษะเดียว: สร้าง customer health score แบบง่ายสำหรับ SaaS ช่วงแรก เพื่อรู้ว่าลูกค้าคนไหนควรช่วย, ควรขยาย, หรือควรหยุดไล่ตาม.

กับดัก

ลูกค้าได้ first value แล้วคุณคิดว่า "โอเค เขาคงอยู่ต่อ" แต่สัปดาห์ถัดมาเขาไม่ส่งข้อมูล ไม่ตอบ ไม่ชวนทีมใช้ และสุดท้ายไม่ต่อ.

เป้าหมาย

มีสัญญาณ 5-6 ตัวที่ตรวจทุกสัปดาห์ แล้วเปลี่ยนเป็น action ที่ชัด: help, recover, expand, หรือ stop.

1. First value ยังไม่ใช่ retention

บทเรียนก่อนเราพาลูกค้าไปถึง value แรก. แต่ใน SaaS ลูกค้าจะไม่อยู่เพราะเคยได้ value หนึ่งครั้ง เขาจะอยู่เพราะ value นั้นเกิดซ้ำในงานจริง. Stripe แยก metrics เรื่อง activation, engagement, retention, churn เพราะแต่ละช่วงตอบคำถามคนละแบบ. Early founder ต้องแปล metrics เหล่านี้เป็นพฤติกรรมลูกค้าที่เห็นได้จริง.

ถ้าลูกค้าไม่กลับมาทำ repeat event รอบที่สอง คุณยังไม่มี retention loop มีแค่ good demo.

2. Health signal ที่ควรดู

SignalHealthyAt risk / Critical
Repeat usage กลับมาใช้/ส่ง input/รัน workflow ตามรอบ ได้ report แรกแล้วไม่มีรอบถัดไป
Outcome action เอา output ไปตัดสินใจหรือแก้งานจริง อ่านแล้วชมว่า interesting แต่ไม่ทำอะไร
Stakeholder มี owner และ budget owner รู้ว่าทำไมต้องใช้ มีแต่ user คนเดียว ไม่มีคนซื้อ/หัวหน้ารับรู้
Friction support ลดลงหลังใช้รอบแรก ทุกครั้งต้องให้คุณช่วยเยอะเท่าเดิม
Next commitment มีนัดรอบถัดไป, next report, renewal, expansion ไม่มีวันที่แน่นอน มีแต่ "เดี๋ยวดูอีกที"

3. Weekly customer success loop

  1. Check usage: รอบนี้ลูกค้าได้ทำ repeat event ไหม
  2. Ask outcome: output ล่าสุดเอาไปทำอะไรจริง
  3. Capture blocker: ถ้าไม่ใช้ซ้ำ ติดคน ข้อมูล เวลา ความเชื่อใจ หรือ value ไม่พอ
  4. Agree next action: นัดวันที่และ action ถัดไปแบบ specific
  5. Score health: Healthy, At risk, หรือ Critical แล้วเลือก response

นี่คือ "do things that don't scale" ในรูปแบบ customer success. คุณยังไม่ต้องมี dashboard health score ใหญ่โต แค่ต้องไม่ปล่อยให้สัญญาณหายไปในความรู้สึก.

4. Churn risk ไม่ได้หน้าตาเหมือนคำด่าเสมอไป

คำพูดที่ฟังดูดีแต่เสี่ยง

  • "น่าสนใจมาก เดี๋ยวลองดู"
  • "ทีมเรายุ่ง เดี๋ยวส่งข้อมูลให้"
  • "ขอดู dashboard เพิ่มก่อน"
  • "ถ้าทำได้ครบกว่านี้น่าจะดี"

พฤติกรรมที่น่าเชื่อกว่า

  • นัดรอบถัดไปทันที
  • ชวนหัวหน้าหรือทีมเข้ามาดู
  • เอา output ไปแก้ process จริง
  • ถามเรื่องราคา, security, rollout

5. แบบฝึกหัด: Customer health score

ให้คะแนนลูกค้า pilot หรือ customer คนหนึ่ง. ตัวประเมินนี้เป็น JavaScript ในไฟล์ ไม่เรียก AI และไม่ส่งข้อมูลออกไปไหน.

รอข้อมูลลูกค้าอยู่.

6. เลือก action หลังได้ score

Healthy

ทำ: ขอ case study, referral, เพิ่ม use case, ชวนทีมเพิ่ม.

At risk

ทำ: นัด recovery call 15 นาที หา blocker จริง และ agree next action ภายในสัปดาห์นี้.

Critical

ทำ: ตัดสินใจว่าจะ save ด้วย concierge help หนึ่งรอบ หรือหยุดไล่ตามถ้า value ไม่พอ.

7. Mini Quiz

สถานการณ์: ลูกค้าได้รับ report แรกแล้วบอกว่า "ดีมาก" แต่ 10 วันผ่านไปไม่ส่งข้อมูลรอบใหม่ และตอบว่า "ช่วงนี้ยุ่งมาก เดี๋ยวกลับมาดู" คุณควรทำอะไร?

เลือกคำตอบหนึ่งข้อ.
