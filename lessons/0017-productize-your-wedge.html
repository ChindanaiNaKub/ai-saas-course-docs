Lesson 0017 · Productize your wedge

เปลี่ยน pilot/manual work ให้เป็น SaaS ที่ขายซ้ำได้

บทเรียนนี้สอนทักษะเดียว: หา wedge ที่ควร productize จากหลักฐานจริง แล้วตัดสิ่งที่ทำให้กลายเป็น custom service ออกจาก v1.

กับดัก

คุณมีลูกค้า pilot หลายคน เริ่มได้เงิน เริ่มมี request แต่ workflow แต่ละรายต่างกันจน product ที่จะ build ไม่ชัด.

เป้าหมาย

เลือก core workflow หนึ่งอันที่คนกลุ่มเดียวกันใช้ซ้ำได้ วัดได้ ขายได้ และมีขอบเขตชัดว่าอะไรยังไม่ทำ.

1. Wedge คือประตูเล็ก ไม่ใช่ product ทั้งจักรวาล

wedge คือ use case แคบๆ ที่ลูกค้ากลุ่มหนึ่งเจ็บจริง จ่ายได้ และใช้ซ้ำได้. มันเล็กพอให้คุณ build ได้เร็ว แต่สำคัญพอให้ลูกค้าเปิดประตู. YC มักย้ำเรื่อง MVP และการทำสิ่งที่ไม่ scale ก่อน เพราะ early product ควรเรียนรู้จากลูกค้าจริง ไม่ใช่เริ่มจาก platform ใหญ่.

Productize ไม่ได้แปลว่า automate ทุกอย่าง. Productize แปลว่าเลือก workflow ที่ซ้ำได้ แล้วทำให้มันเชื่อถือได้.

2. Service, tool, หรือ SaaS?

หน้าตาสัญญาณความเสี่ยง
Custom service ทุกลูกค้าต้องปรับ logic, report, data format, และ success criteria ใหม่ รายได้มา แต่ scale ด้วยคน ไม่ใช่ product
Useful tool ลูกค้าใช้ได้ แต่ไม่มี repeat trigger หรือ budget owner ชัด อาจเป็น feature ไม่ใช่ business
Productized wedge ICP เดียวกัน, input คล้ายกัน, output คล้ายกัน, repeat event คล้ายกัน, pricing ผูกกับ value ได้ เริ่มกลายเป็น SaaS ที่ขายซ้ำได้

3. Productizable wedge มี 6 สัญญาณ

Same buyer

คนซื้อ/คนใช้เป็น persona เดียวกัน ไม่ใช่ลูกค้าคนละโลก.

Same trigger

ปัญหาเกิดจากเหตุการณ์ซ้ำ เช่น weekly report, monthly reconciliation, missed leads.

Same input

ข้อมูลที่ต้องใช้มีรูปแบบคล้ายกันพอทำ checklist/template ได้.

Same output

ผลลัพธ์ที่ลูกค้าต้องการมี structure ซ้ำ ไม่ใช่ custom analysis ใหม่ทุกครั้ง.

Same action

ลูกค้าเอา output ไปทำ action คล้ายกัน จึงวัด value ได้.

Same price logic

ราคาอธิบายด้วย value driver เดียวกัน เช่น volume, seats, reports, saved hours, recovered revenue.

4. Wedge boundary สำคัญกว่า feature list

ถ้าไม่มี boundary คุณจะรับทุก request แล้ว product จะบวม. Boundary ที่ดีบอกทั้งสิ่งที่ทำและไม่ทำ.

ตัวอย่าง boundary ที่คม

"เราช่วย ecommerce SME หา lost orders จากแชท LINE/Facebook รายสัปดาห์ แล้วส่ง action list ให้ทีมตอบแชท."

ตัวอย่างที่กว้างเกินไป

"AI dashboard สำหรับธุรกิจออนไลน์ ใช้วิเคราะห์ทุกอย่างจากทุกช่องทาง และช่วยตัดสินใจให้ดีขึ้น."

5. แบบฝึกหัด: Productizable wedge score

กรอกหลักฐานจาก pilot/manual work. ตัวประเมินนี้เป็น JavaScript ในไฟล์ ไม่เรียก AI และไม่ส่งข้อมูลออกไปไหน.

รอหลักฐาน wedge อยู่.
