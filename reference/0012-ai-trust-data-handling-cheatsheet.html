AI Trust and Data Handling Cheat Sheet

ใช้ก่อนขอข้อมูลจริงจากลูกค้า โดยเฉพาะแชท, ไฟล์, ข้อมูลลูกค้าปลายทาง, หรือข้อมูลธุรกิจลับ.

แก่นคิด

Trust ไม่ใช่หน้า privacy policy ยาวๆ. Trust คือขอข้อมูลให้น้อยที่สุด อธิบายให้ชัด ใช้ตามที่สัญญา ลบตามเวลา และป้องกัน output/LLM risk.

หลัก data minimization คือเก็บเฉพาะข้อมูลที่จำเป็นกับ purpose นั้น และเก็บเท่าที่จำเป็น. นี่เป็นหลักสำคัญทั้งใน GDPR-style privacy และการออกแบบ AI pilot ที่ลูกค้ากล้าให้ข้อมูล.

Data trust checklist

  1. Purpose: ขอข้อมูลไปทำอะไร
  2. Minimum data: ต้องใช้ข้อมูลน้อยสุดแค่ไหน
  3. Redaction: ตัดชื่อ, เบอร์, email, address, payment, medical หรือ sensitive fields ได้ไหม
  4. Storage: เก็บที่ไหน ใครเข้าถึงได้
  5. Retention: ลบเมื่อไหร่
  6. Model provider: ส่งข้อมูลไป provider ไหน และ provider ใช้ข้อมูลยังไง
  7. Output risk: มี human review หรือ validation ก่อนลูกค้าใช้ตัดสินใจไหม

AI-specific risks

RiskเกิดยังไงGuardrail
Prompt injection ข้อมูลลูกค้ามีคำสั่งแฝงหลอก model แยก system instructions, จำกัด tool access, validate output
Sensitive disclosure model เผยข้อมูลที่ไม่ควรถูกส่งต่อ redaction, access control, output filtering
Overreliance ลูกค้าเชื่อ output ที่ผิดโดยไม่มี review confidence labels, human review, disclaimers for decisions
Data retention creep เก็บทุกอย่างไว้เผื่อใช้ทีหลัง retention window, deletion workflow, purpose limitation
