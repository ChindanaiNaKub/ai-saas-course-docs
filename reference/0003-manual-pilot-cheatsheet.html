Manual Pilot Cheat Sheet

ใช้หลังมี customer interview signal แล้ว แต่ยังไม่ควรสร้าง SaaS เต็มตัว. เป้าหมายคือขาย/ส่งมอบผลลัพธ์แบบ manual เพื่อเรียนรู้ก่อน automate.

แก่นคิด

อย่าเริ่มจาก "เราจะ build feature อะไร" ให้เริ่มจาก "เราจะส่งมอบผลลัพธ์อะไรให้ลูกค้า 1 ราย ภายใน 7 วัน".

MVP ตามแนว Lean Startup คือวิธีเก็บ validated learning ด้วย effort ต่ำที่สุด ไม่ใช่ product เต็มที่ตัด feature ออกนิดหน่อย.

Manual pilot 7 ช่อง

  1. Customer: ลูกค้าคนแรกเป็นใคร ชื่อ/ธุรกิจ/role ชัดไหม
  2. Promise: ผลลัพธ์ที่รับปากใน 7 วันคืออะไร
  3. Input: ต้องขอข้อมูลอะไรจากลูกค้า
  4. Manual workflow: เราจะทำเองด้วย spreadsheet, AI chat, script, หรือ service ยังไง
  5. Delivery: ส่งงานเป็น report, dashboard mock, file, LINE message, หรือ call แบบไหน
  6. Commitment: ลูกค้าจ่าย/มัดจำ/ส่งข้อมูล/นัด follow-up/แนะนำคนอื่นไหม
  7. Learning goal: ต้องเรียนรู้อะไรถึงจะตัดสินใจ build หรือ kill

ตัวอย่าง

IdeaManual pilotห้าม build ก่อน
AI สรุปแชท LINE ร้านค้า ให้ร้าน export/forward แชท 100 รายการ แล้วคุณทำ summary + lost sales report ด้วย AI/manual ภายใน 24 ชม. LINE integration, auth, dashboard, billing
CRM สำหรับคลินิก รับ spreadsheet นัดหมาย 1 สัปดาห์ แล้วส่ง reminder list + no-show analysis ให้เจ้าของคลินิก ระบบ booking เต็ม, staff roles, patient portal
AI ช่วย agency ทำ report รับ campaign data 1 ลูกค้า แล้วทำ report draft + insight 5 ข้อให้ account manager multi-client dashboard, template builder, API sync

Build / continue / kill

Build small automation เมื่อ...

  • ลูกค้าใช้ผลลัพธ์จริง
  • เขายอมจ่ายหรือมี budget path
  • งาน manual ซ้ำชัดและ automate ได้
  • มีลูกค้าคล้ายกันอย่างน้อย 3 รายสนใจ

Kill หรือเปลี่ยน segment เมื่อ...

  • ลูกค้าชมแต่ไม่ส่งข้อมูล/ไม่นัดต่อ
  • ผลลัพธ์ไม่ถูกใช้ใน workflow จริง
  • ต้นทุนส่งมอบสูงกว่า value มาก
  • ทุกคนต้อง custom ต่างกันจนเป็น agency work
