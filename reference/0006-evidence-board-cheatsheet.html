Evidence Board Cheat Sheet

ใช้หลังทำ interviews, manual pilot, paid offer, และ distribution sprint. เป้าหมายคือเลิกตัดสินใจจากอารมณ์ แล้วตัดสินจาก evidence.

แก่นคิด

อย่า build เพราะ "รู้สึกว่ามีหวัง" และอย่า kill เพราะ "โดนปฏิเสธแล้วเจ็บ". ให้ดูว่าสัญญาณอ่อนหรือแข็งตรงไหน.

YC ย้ำเรื่อง launch, talk to users, build what people want. Lean Startup ย้ำ validated learning. Evidence board คือวิธีรวมการเรียนรู้เหล่านั้นให้อยู่บนหน้าเดียว.

Evidence 4 ด้าน

ด้านสัญญาณแข็งสัญญาณอ่อน
Customer pain เหตุการณ์ล่าสุด, cost ชัด, workaround มีอยู่แล้ว คำชม, opinion, "น่าสนใจ"
Outcome/pilot เขาใช้ output ใน workflow จริง, ขอใช้ต่อ ดู demo แล้วเฉย, ไม่ส่งข้อมูลจริง
Money จ่าย, มัดจำ, ขอ invoice, พา budget owner มา ขอใช้ฟรีแบบไม่มี commitment
Distribution หา lead ซ้ำได้, reply, calls, paid pilots ไม่มี reply, list หาไม่ได้, channel ไม่ถึงลูกค้า

Decision rules

Build small automation

  • paid/serious pilots มีอย่างน้อย 2-3 ราย
  • pain + outcome ซ้ำใน segment เดียวกัน
  • distribution channel หา calls ได้ซ้ำ
  • manual workflow มีส่วนที่ซ้ำและ automate ได้

Iterate

  • pain มีจริง แต่ offer ยังไม่ convert
  • reply มี แต่ไม่มี paid pilot
  • pilot มี value แต่ segment กว้างหรือ channel ยังมั่ว

Pivot

  • pain จริง แต่คนจ่ายไม่ใช่ segment ที่คิด
  • use case หนึ่งใน pilot สำคัญกว่า idea เดิม
  • channel หนึ่งชัดกว่า product angle เดิม

Kill

  • ไม่มี recent pain หลังคุยหลายคน
  • ไม่มีใครส่งข้อมูล/จ่าย/นัดต่อ
  • ต้นทุนส่งมอบสูงกว่า value และแก้ไม่ออก
  • lead หาไม่ได้ซ้ำแม้ลองหลาย channel อย่างมีระบบ

คำถามก่อน build

  1. เรากำลัง automate workflow ที่ลูกค้าใช้จริงหรือแค่ demo?
  2. ลูกค้าที่จ่ายมีอะไรเหมือนกัน?
  3. ส่วนไหนใน manual pilot ซ้ำที่สุดและเจ็บที่สุด?
  4. ถ้า build 7 วัน ควร build เฉพาะอะไรที่ลด friction ใน pilot?
  5. ถ้าไม่ build อะไรเลย เรายังขาย pilot ได้อีกไหม?
