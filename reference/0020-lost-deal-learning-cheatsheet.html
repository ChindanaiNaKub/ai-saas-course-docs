Lost Deal Learning Cheat Sheet

ใช้หลัง sales call/proposal ที่ไม่ close. เป้าหมายคือหา root cause จริง ไม่ใช่ react ตาม objection แรกที่ prospect พูด.

แก่นคิด

ทุกดีลที่แพ้ต้องให้ learning หนึ่งข้อและ experiment ถัดไปหนึ่งอย่าง.

คำว่า "แพงไป" อาจหมายถึง price จริง หรืออาจหมายถึง value, urgency, trust, buyer fit ยังไม่ชัด.

Lost reason categories

CategorySignalNext experiment
ICP fit ไม่ใช่ buyer/workflow/budget ที่ package ออกแบบมา ปรับ targeting และ qualification
Pain weak ไม่มี cost, workaround, urgency ชัด ถาม past behavior และหา segment ที่เจ็บกว่า
Trust gap กังวล proof, accuracy, data, security เพิ่ม proof, pilot, case study, trust docs
Urgency gap เห็นด้วยแต่เลื่อนเรื่อยๆ หา trigger/deadline หรือเปลี่ยน segment
Package mismatch outcome ใช่ แต่ scope/usage/onboarding ไม่ fit ปรับ package ladder
Price/value gap ROI/budget ไม่ผ่านแม้เข้าใจ value ปรับ value metric, price metric, buyer segment
Timing fit ดีแต่ติดงบ/season/internal priority ตั้ง follow-up date และ nurture proof

Post-loss questions

  1. ถ้าต้องเลือกเหตุผลเดียวที่ยังไม่เดินต่อ คืออะไร?
  2. ปัญหานี้มี cost จริงเท่าไรในเดือนที่ผ่านมา?
  3. ใครต้องเห็นหรืออนุมัติก่อน?
  4. อะไรที่ยังไม่ trust: proof, accuracy, data, security, delivery?
  5. ถ้ากลับมาคุยอีกครั้ง ควรเป็น trigger หรือช่วงเวลาไหน?

Friday review

  • 3+ ICP losses: lead source/positioning ผิด
  • 3+ pain weak losses: segment ไม่เจ็บพอ หรือ discovery ยังไม่ลึก
  • 3+ trust losses: ต้องเพิ่ม proof, data handling, case study
  • 3+ price losses: เช็ก ROI, price metric, buyer budget
  • 3+ timing losses: หา trigger/event หรือ nurture sequence
