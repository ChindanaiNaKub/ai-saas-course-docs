Customer Health Cheat Sheet

ใช้หลังลูกค้าได้ first value แล้ว. เป้าหมายคือจับสัญญาณ churn ให้เร็วพอที่จะช่วยได้ และจับสัญญาณ healthy ให้เร็วพอที่จะขยายได้.

แก่นคิด

First value คือจุดเริ่ม. Retention เกิดเมื่อ value นั้นเกิดซ้ำใน workflow จริงของลูกค้า.

อย่าวัดแค่คำชม. วัดพฤติกรรมที่มี cost: ส่งข้อมูลซ้ำ, นัดรอบถัดไป, ชวน stakeholder, ใช้ output ตัดสินใจ, ต่ออายุ, เพิ่มทีม.

Weekly health loop

  1. Usage: repeat event รอบนี้เกิดไหม
  2. Outcome: output ถูกใช้ทำ action อะไร
  3. Stakeholder: owner และ budget owner ยัง engaged ไหม
  4. Friction: ลูกค้าติดอะไรซ้ำๆ หรือ support ลดลงแล้ว
  5. Commitment: next step มีวันและเจ้าของชัดไหม

Health score แบบเร็ว

สถานะสัญญาณAction
Healthy ใช้ซ้ำ, output ถูก action, owner ชัด, friction ต่ำ, มี next commitment ขอ case study/referral, เพิ่ม use case, ชวนทีมเพิ่ม, คุย expansion
At risk ใช้ซ้ำไม่สม่ำเสมอ, ยังไม่ action ชัด, next step หลวม, owner ไม่ครบ นัด recovery call 15 นาที หา blocker และล็อก next action ในสัปดาห์นี้
Critical ไม่มี repeat event, เงียบเกิน 7 วัน, ไม่มี owner, support burden สูง, ไม่มี next step ช่วย concierge หนึ่งรอบถ้ามี value จริง หรือหยุดไล่ตามถ้า pain/budget ไม่พอ

Recovery call script

  1. "รอบแรกได้อะไรที่มีประโยชน์จริงบ้าง?"
  2. "ทำไมรอบถัดไปยังไม่เกิด: ข้อมูล, เวลา, trust, คนอนุมัติ, หรือ value ยังไม่พอ?"
  3. "ถ้าผมช่วยรันรอบถัดไปให้ภายใน 24 ชม. คุณจะเอาผลลัพธ์ไปทำอะไร?"
  4. "ใครต้องเห็นผลลัพธ์นี้เพื่อให้ตัดสินใจต่อได้?"
  5. "เราล็อกวันรอบถัดไปเลยไหม: วันไหน เวลาไหน ใครส่ง input?"

Founder decision

Save

ลูกค้าเห็น value แต่ติด friction. ช่วยแบบ manual หนึ่งรอบ แล้ว automate friction ที่ซ้ำ.

Expand

ลูกค้าใช้ซ้ำและมี stakeholder. ขอ use case ถัดไป, referral, หรือเพิ่มทีม.

Stop

ไม่มี pain ชัด, ไม่มี owner, ไม่ใช้ซ้ำ, หรือ support สูงแต่ value ต่ำ. อย่าปล่อยให้เคสนี้กินเดือนของคุณ.
