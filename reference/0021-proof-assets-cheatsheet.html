Proof Assets Cheat Sheet

ใช้เมื่อ sales calls เจอ objection ซ้ำ. เป้าหมายคือสร้างหลักฐานที่ตอบความเสี่ยงซ้ำๆ แทนการอธิบายสดทุกครั้ง.

แก่นคิด

ถ้าคำถามเดิมเกิดซ้ำ 3 ครั้ง ให้สร้าง proof asset สั้นๆ ที่ใช้ซ้ำใน pipeline.

Asset map

ObjectionProof assetควรมีอะไร
ช่วยได้จริงไหม Result proof before/after, quote, case, customer action
คุ้มราคาไหม ROI proof saved hours, recovered revenue, avoided mistakes
ข้อมูล/AI เชื่อได้ไหม Trust proof data minimization, redaction, retention, provider, limits
เริ่มแล้วเสี่ยงไหม Pilot proof timeline, inputs, success criteria, decision gate
setup ยุ่งไหม Implementation proof checklist, roles, timeline, sample output

Proof snippet template

For [customer type], before [pain]. We did [workflow]. Result: [evidence]. Risk control: [trust note].

First proof kit

  1. Mini case: ใคร, pain, workflow, result, next action
  2. Pilot plan: 7-14 วัน, inputs, deliverables, success criteria
  3. Trust note: data received, redaction, retention, AI provider, limitations

Do not overclaim

  • อย่าใช้ตัวเลขที่ยังไม่ได้วัด
  • อย่าบอกว่า secure/private โดยไม่มีรายละเอียด
  • อย่าใช้ case จากคนละ ICP เพื่อหลอกว่า fit
  • อย่า promise outcome ที่คุณควบคุมไม่ได้
