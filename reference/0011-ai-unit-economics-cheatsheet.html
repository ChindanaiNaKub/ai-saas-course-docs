AI Unit Economics Cheat Sheet

ใช้ก่อนตั้งราคา subscription หรือ usage-based pricing ให้ AI SaaS. เป้าหมายคือไม่ให้ธุรกิจยิ่งโตยิ่งขาดทุน.

แก่นคิด

AI cost คือ cost of goods sold แบบใหม่. ถ้าไม่วัดต่อ action/customer คุณอาจขายได้ แต่ margin ติดลบ.

Stripe แนะนำให้ SaaS ดู metrics เช่น gross margin, CAC, churn และ expansion. สำหรับ AI SaaS ต้องเพิ่ม cost per outcome เข้าไปด้วย เพราะ inference, storage, retrieval, และ human review อาจโตตาม usage.

สูตรพื้นฐาน

Gross margin = (price - variable cost) / price

Variable cost = AI calls + storage + third-party APIs + human review + support per customer

Safe price floor = variable cost / (1 - target gross margin)

ต้นทุนที่มักลืม

Costตัวอย่างวิธีคุม
AI inference tokens, images, embeddings, reranking, agent steps cache, cheaper model, limit runs, batch jobs
Human review ตรวจ report, แก้ output, onboarding call templates, QA checklist, confidence thresholds
Support LINE chat, setup, bug reports, custom requests scope limits, docs, paid onboarding
Data handling storage, sync, exports, privacy work retention policy, file limits, structured imports
Distribution outreach time, ads, partner rev-share track CAC by channel

Pricing model choices

Fixed subscription

ดีเมื่อ usage คาดเดาได้. ต้องมี fair-use limit ไม่งั้น heavy users กิน margin.

Usage-based

ดีเมื่อ cost โตตาม usage ชัด เช่น report/run/message. ต้องอธิบาย value ให้เข้าใจง่าย.

Hybrid

base fee + usage. เหมาะกับ AI SaaS ระยะแรก เพราะมี revenue floor และป้องกัน overuse.

Outcome-based

ดีเมื่อวัดผลลัพธ์ได้และลูกค้าเชื่อ attribution. ยากขึ้นแต่ align value สูง.

AI margin guardrails

  • ดี: gross margin 70%+ หลัง variable cost และ support พื้นฐาน
  • ระวัง: margin 40-70% ต้องคุม usage และ support ให้ดี
  • อันตราย: margin ต่ำกว่า 40% หรือ heavy users ทำให้ขาดทุน
